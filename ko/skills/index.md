# 스킬 인덱스

이 폴더는 재사용 가능한 작업 플레이북을 저장합니다. 이 파일은 저장소의
스킬 목록을 관리하는 인덱스입니다.

기본 템플릿에는 이 인덱스만 포함합니다. 반복되는 작업 흐름이 생기면 하위
프로젝트에서 실제 스킬 문서를 추가합니다.

## 현재 스킬

아직 프로젝트 전용 스킬은 없습니다.

스킬을 추가하면 아래 예시 행을 실제 스킬로 교체합니다.

| 스킬 | 경로 | 사용 조건 |
| --- | --- | --- |
| 예시 스킬 | `skills/<skill-name>/skill.md` | 예시 트리거 조건 |

## 언제 스킬을 만들까

같은 지침, 체크리스트, 명령 순서, 검증 루틴을 반복해서 붙여넣고 있다면
스킬로 분리합니다.

좋은 스킬 범위:

- 블로그 글 발행
- Pull Request 리뷰
- PDF 문서 준비
- 읽지 않은 이메일 triage
- 특정 프로젝트 배포

너무 넓은 범위:

- 모든 프로젝트 작업 처리
- 모든 문서 처리
- 모든 외부 서비스 관리

## 권장 스킬 구조

```text
skills/
└── skill-name/
    └── skill.md
```

권장 내용 구조:

```markdown
# 스킬 이름

## 목적

이 스킬이 에이전트에게 무엇을 하게 하는지 설명합니다.

## 사용 조건

- 트리거 조건 1
- 트리거 조건 2

## 입력

- 필요한 파일, URL, 명령, 인증 정보, 사용자 결정

## 절차

1. 관련 파일을 읽습니다.
2. 작업 흐름을 수행합니다.
3. 결과를 검증합니다.
4. 변경 파일과 남은 문제를 보고합니다.

## 검증

- 결과 확인에 사용할 명령 또는 체크리스트

## 실패 대응

- 의존성, 인증 정보, 명령이 없을 때 어떻게 처리할지
```

## 규칙

- 각 스킬은 작고 명확하게 유지합니다.
- 예시는 짧게 둡니다.
- 개인 인증 정보는 `.env` 같은 로컬 파일에 두고 스킬 문서에 적지 않습니다.
- 가능하면 결정적 검사를 포함합니다.
- 스크립트는 기본 템플릿이 아니라 하위 프로젝트에서만 추가합니다.
- 스킬을 추가, 이름 변경, 삭제하거나 트리거를 바꿀 때마다 이 인덱스를
  갱신합니다.
- 서브에이전트는 사용자의 실제 업무 흐름에 맞게 설정합니다. 특정 스킬을
  전문 에이전트에게 위임해야 한다면 권장 모델, 책임 범위, 위임 조건을
  해당 스킬 또는 이 인덱스에 적습니다.
