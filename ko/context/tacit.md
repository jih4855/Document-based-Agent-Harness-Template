# 암묵지 인덱스

이 파일은 하네스에 지속적으로 적용할 운영 지식을 저장합니다.

짧게 유지합니다. 반복 절차가 길어지면 별도 스킬 문서로 옮깁니다.

## 응답 방식

- 직접적이고 구체적으로 답합니다.
- 추상 조언보다 실행 가능한 단계를 우선합니다.
- 한 경로를 선택할 때는 tradeoff를 설명합니다.
- 프로젝트 사실을 지어내지 않습니다. 먼저 파일을 읽습니다.

## 작업 방식

- 저장소의 기존 구조를 우선합니다.
- 변경 범위를 작게 유지합니다.
- 관련 없는 리팩터링을 피합니다.
- 자동화보다 Markdown 플레이북을 먼저 만듭니다.
- 예시 파일은 템플릿이며 live state가 아닙니다.

## 맥락 위생

- 의미 있는 결정 뒤에는 `context/memory.md`를 갱신합니다.
- 작업 상태가 바뀌면 `context/task.md`를 갱신합니다.
- 비밀 정보와 개인 데이터는 추적되는 맥락 파일에 넣지 않습니다.
- 긴 로그, 생성 산출물, 로컬 메모는 Git에서 제외합니다.
- 이 파일을 최신 암묵지 인덱스로 유지합니다. 세부 tacit 문서를 추가로
  분리하면 즉시 이 파일에 링크와 역할을 반영합니다.

## 암묵지 업데이트 규칙

- 지속될 선호, 반복 실패 패턴, 성공한 우회 방법, 워크플로우 결정을
  명확해진 시점에 기록합니다.
- 임시 작업 상태는 여기에 기록하지 않습니다. `context/task.md`나
  `context/memory.md`를 사용합니다.
- 더 이상 맞지 않는 규칙은 삭제하거나 낡은 규칙으로 표시합니다.

## 스킬 작성

- 하나의 스킬은 하나의 작업 흐름만 다룹니다.
- 스킬 이름은 도구 이름보다 행동을 설명해야 합니다.
- 트리거, 입력, 절차, 검증, 실패 대응을 포함합니다.
- 스크립트는 기본 템플릿이 아니라 하위 프로젝트에서 필요할 때만 추가합니다.
- 스킬을 추가, 이름 변경, 삭제하거나 트리거를 바꾸면 `skills/index.md`를
  갱신합니다.
- 서브에이전트 권장은 사용자의 실제 업무 환경에 맞을 때만 문서화합니다.

## 완료 규칙

- 변경 파일을 보고합니다.
- 검증 결과를 보고합니다.
- 남은 위험이나 미검증 부분을 보고합니다.
- 이후 작업에도 적용될 교훈만 이 파일에 기록합니다.
