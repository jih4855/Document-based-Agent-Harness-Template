# 문서 기반 에이전트 하네스 템플릿

코딩 에이전트와 업무 자동화 에이전트를 위한 최소 문서형 하네스입니다.

이 템플릿은 앱, 패키지, 런타임을 포함하지 않습니다. 대신 에이전트가
맥락을 읽고, 로컬 규칙을 따르고, 스킬을 활용하고, 다음 세션을 위한
상태를 남길 수 있도록 반복 가능한 폴더 구조와 Markdown 계약을 제공합니다.

## 이 템플릿의 목적

에이전트 하네스는 LLM 기반 작업 흐름을 감싸는 제어 레이어입니다.
이 템플릿에서는 그 제어 레이어를 문서로 표현합니다.

- `agent.md`는 워크스페이스 운영 계약을 정의합니다.
- `context/tacit.md`는 지속적으로 적용할 규칙, 선호, 교훈을 저장합니다.
- `context/memory.example.md`는 세션 간 상태 기록 방식을 보여줍니다.
- `context/task.example.md`는 현재 작업을 추적하는 방식을 보여줍니다.
- `skills/index.md`는 사용 가능한 스킬 목록과 재사용 가능한 스킬 문서 작성법을 정의합니다.

커스텀 SDK나 CLI를 만들기 전에, 에이전트가 매 세션 일관되게 움직이도록
만들고 싶을 때 사용합니다.

## 폴더 구조

```text
.
├── README.md
├── agent.md
├── .gitignore
├── context/
│   ├── tacit.md
│   ├── memory.example.md
│   └── task.example.md
└── skills/
    └── index.md
```

## 빠른 시작

1. 이 템플릿으로 새 저장소를 만듭니다.
2. 예시 상태 파일을 실제 상태 파일로 복사합니다.

```bash
cp context/memory.example.md context/memory.md
cp context/task.example.md context/task.md
```

3. 사용하는 에이전트 도구에서 저장소를 엽니다.
4. 에이전트에게 먼저 `agent.md`를 읽으라고 지시합니다.
5. 프로젝트별 규칙은 `context/tacit.md`에 추가합니다.
6. 반복되는 작업 흐름은 `skills/` 아래에 Markdown 문서로 추가합니다.
7. 스킬을 추가, 이름 변경, 삭제할 때마다 `skills/index.md`를 갱신합니다.

## 도구별 참고

에이전트 도구마다 프로젝트 지침을 읽는 방식이 다릅니다.

- 도구가 `AGENTS.md`를 자동으로 읽는다면 `agent.md`를 `AGENTS.md`로 복사하거나 심볼릭 링크를 만듭니다.
- 도구가 `CLAUDE.md`를 자동으로 읽는다면 `agent.md`를 `CLAUDE.md`로 복사하거나 심볼릭 링크를 만듭니다.
- 도구가 커스텀 스킬을 지원한다면 `skills/` 폴더를 해당 도구의 스킬 폴더 규칙에 맞게 옮기거나 변환합니다.

이 템플릿은 특정 도구에 종속되지 않도록 중립적인 문서 구조를 유지합니다.

## 설계 원칙

- 지침은 짧고 명확하고 실행 가능하게 작성합니다.
- 자동화 스크립트를 만들기 전에 Markdown 플레이북으로 먼저 정리합니다.
- 개인 상태는 추적하지 않는 `context/memory.md`, `context/task.md`에 둡니다.
- 비밀키, 토큰, 개인 일정, 고객 데이터는 커밋하지 않습니다.
- 의미 있는 결정이 생기면 다음 세션이 이어받을 수 있도록 memory를 갱신합니다.

## 권장 스킬 구조

나중에 실제 스킬을 추가할 때는 아래 구조를 권장합니다.

```text
skills/
└── my-skill/
    └── skill.md
```

각 스킬은 하나의 작업 흐름에 집중합니다. Markdown 플레이북만으로 부족할 때만
개별 프로젝트 안에서 스크립트를 추가합니다.
