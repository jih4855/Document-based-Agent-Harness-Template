# 에이전트 하네스 지침

이 파일은 이 저장소에서 작업하는 에이전트를 위한 기본 운영 계약입니다.
변경 작업을 시작하기 전에 먼저 읽습니다.

## 목적

이 저장소는 문서 기반 에이전트 하네스입니다. 에이전트의 작업 규칙, 작업 상태,
메모리, 재사용 가능한 스킬을 plain Markdown으로 관리해 워크플로우를 쉽게
검사하고 복사하고 수정할 수 있게 합니다.

## 필수 읽기 순서

세션 시작 시 아래 파일을 순서대로 읽습니다.

1. `agent.md`
2. `context/tacit.md`
3. `context/agent_index.md`
4. `context/task.md`가 있으면 읽고, 없으면 `context/task.example.md`를 읽습니다.
5. `context/memory.md`가 있으면 읽고, 없으면 `context/memory.example.md`를 읽습니다.
6. `skills/index.md`
7. 현재 작업과 직접 관련 있는 스킬 문서

`@context/tacit.md` 같은 include 문법에 의존하지 않습니다. 일부 런타임은 include를
펼치지 않습니다. 필요한 파일은 직접 읽습니다.

## 작업 사이클

작지 않은 작업은 아래 흐름을 따릅니다.

1. 관련 맥락을 읽습니다.
2. 단계가 여러 개인 작업이면 즉시 실행할 계획을 짧게 밝힙니다.
3. 가장 작고 유용한 변경부터 수행합니다.
4. 적절한 명령, 검사, 체크리스트로 결과를 검증합니다.
5. 결정이나 지속될 사실이 바뀌면 `context/memory.md`를 갱신합니다.
6. 작업 상태가 바뀌면 `context/task.md`를 갱신합니다.

## 맥락 규칙

- `context/tacit.md`는 지속 규칙, 선호, 관례, 교훈을 저장합니다.
- `context/agent_index.md`는 반복 업무를 문서화된 런타임 에이전트로 라우팅합니다.
- `context/memory.md`는 최근 세션 간 상태를 저장합니다.
- `context/task.md`는 현재 작업과 다음 행동을 저장합니다.
- 예시 파일은 템플릿입니다. `*.example.md`를 실제 상태 파일로 취급하지 않습니다.
- 실제 맥락 파일은 짧게 유지합니다. 오래된 세부 내용은 archive로 옮깁니다.

## 암묵지 업데이트 규칙

- 지속될 선호, 운영 규칙, 반복 실패, 성공한 우회 방법이 드러나면
  `context/tacit.md`에 기록합니다.
- 짧게 지나가는 작업 상태는 tacit에 넣지 않습니다. 현재 작업은
  `context/task.md`, 최근 프로젝트 상태는 `context/memory.md`에 둡니다.
- `context/tacit.md`를 최신 암묵지 인덱스로 유지합니다. 나중에 세부 규칙을
  다른 context 파일로 분리하면 즉시 이 인덱스를 갱신합니다.

## 위임 규칙

- 메인 에이전트는 사용자와 전략, 선호, 운영 기준, tacit, skill, 위임 구조를
  조율하고 갱신합니다.
- 서브에이전트는 문서화된 반복 업무를 처리하는 런타임 실행자입니다. 한 번에
  하나의 업무, 필요한 최소 맥락, `context/agent_index.md`에 정의된 공통 출력
  계약만 전달합니다.
- 서브에이전트가 맥락, 권한, 인증, 판단 기준 때문에 막히면 메인이 멘토링,
  좁은 재지시, 직접 인수 중 하나로 처리합니다.
- 정책 변경, 취향 조율, 새 workflow 설계는 사용자가 명시적으로 요청하지 않는 한
  서브에이전트에 넘기지 않습니다.

## 스킬 규칙

- 스킬은 특정 작업 흐름을 위한 재사용 Markdown 플레이북입니다.
- 스킬을 만들거나 수정하기 전에는 `skills/index.md`를 읽습니다.
- 큰 범용 스킬보다 하나의 목적에 집중한 작은 스킬을 선호합니다.
- 스킬에는 명령, 파일, 검증, 판단 기준을 적을 수 있습니다.
- 이 기본 템플릿에는 실행 스크립트를 추가하지 않습니다. 필요한 스크립트는
  하위 프로젝트에서 별도로 둡니다.
- 스킬과 서브에이전트는 사용자의 실제 업무 상황에 맞게 설정합니다. 이
  템플릿은 구조만 제공하며, 어떤 도구, 모델, 전문 에이전트를 써야 하는지
  미리 가정하지 않습니다.
- 스킬을 추가, 이름 변경, 삭제하거나 트리거를 바꾸면 `skills/index.md`를
  최신 상태로 갱신합니다.

## 안전 규칙

- `.env`, API 키, OAuth 토큰, 고객 데이터, 개인 일정은 커밋하지 않습니다.
- 부작용이 있는 명령을 실행하기 전에 대상 경로와 예상 결과를 확인합니다.
- 되돌리기 어려운 작업 전에는 dry-run 또는 읽기 전용 검사를 먼저 합니다.
- 외부 인증 정보나 파괴적 작업이 필요하면 먼저 사용자 확인을 받습니다.

## 완료 체크리스트

작업 완료를 보고하기 전에 아래를 확인합니다.

- 무엇이 바뀌었는지 확인합니다.
- 무엇으로 검증했는지 확인합니다.
- 검증하지 못한 부분을 명시합니다.
- 작업 상태나 지속 교훈이 생겼다면 live context 파일을 갱신합니다.
